# Public Repository Checklist

Use this before publishing or sharing a small open-source repository.

## Required Basics

- [ ] `README.md` explains purpose, install, and usage.
- [ ] `LICENSE` exists.
- [ ] `.gitignore` excludes local, generated, and private files.
- [ ] CI runs tests or at least a preflight check.
- [ ] Security contact or `SECURITY.md` is present.

## Privacy and Safety

- [ ] No `.env`, keys, tokens, or credentials.
- [ ] No personal email, phone, customer, or private work data.
- [ ] Sample data is clearly synthetic.
- [ ] Generated reports are reviewed before publishing.

## Maintainer Workflow

- [ ] First issues or roadmap are visible.
- [ ] Release notes or tags exist.
- [ ] Contribution expectations are documented.
- [ ] Automation checks are easy for another maintainer to run.
