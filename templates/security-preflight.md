# Security Preflight

Review these before publishing a repository or accepting a contribution.

- [ ] Secrets are not present in files or history.
- [ ] Risky file names such as `.env`, `credentials.json`, or private keys are absent.
- [ ] Sample data does not contain real people, customers, or internal records.
- [ ] Large generated files are intentionally included or moved to release assets.
- [ ] CI blocks high-severity preflight findings.
- [ ] Security guidance is documented for contributors.
